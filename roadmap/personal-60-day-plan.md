# 个人版 60 天模型部署学习路线

## 你的当前情况

- 年级：大三，电子信息工程。
- 时间：每天 7 小时。
- 基础：Python 入门，C++ 低于入门但有 C 语言基础，Linux 基本没有，深度学习入门。
- 目标：暑假结束后准备秋招和实习，不考研，至少做出 2 个就业相关项目。
- 硬件：准备购买 Jetson Orin Nano，原本想做视觉小车。
- 竞赛：关注华为 ICT，也想尝试天池和 Kaggle。

## 推荐主方向

主方向：边缘视觉部署。

原因：

- 和电子信息工程、Jetson、视觉小车最匹配。
- 比纯视觉算法更容易在两个月内做出可展示成果。
- 能覆盖模型部署、TensorRT、Linux、摄像头、边缘设备、性能优化等岗位关键词。
- 后续可以投边缘 AI、模型部署、视觉工程化、嵌入式 AI、机器人感知部署相关岗位。

暂时不建议一开始走太底层的 CUDA 算子开发或纯嵌入式驱动方向，因为这两类对 C++、Linux 和系统基础要求更高，短期内投入产出比不如边缘视觉部署。

## 每日 7 小时节奏

| 时间块 | 内容 | 目标 |
| --- | --- | --- |
| 1.5 小时 | 基础学习 | Linux / C++ / 深度学习基础轮换 |
| 1.5 小时 | 跟做实验 | 跑通教程、复现小 demo |
| 2 小时 | 项目开发 | 围绕两个作品集项目写代码 |
| 1 小时 | 性能测试与排错 | 记录 FPS、延迟、显存、错误 |
| 1 小时 | 复盘与 GitHub | 写日志、提交代码、整理 README |

每天必须留下一个可见产出：一段代码、一张表、一份笔记、一条排错记录、一个截图或一次 Git 提交。

## 两个就业项目

### 项目 1：PC 端模型部署与推理优化

题目建议：YOLO 目标检测模型的 ONNX/TensorRT 部署与性能优化。

交付内容：

- PyTorch / ONNX Runtime / TensorRT 三种推理方式。
- FP32 / FP16 / INT8 对比。
- 延迟、FPS、显存占用对比表。
- 完整 README、运行脚本、测试图片和结果图。

这个项目用于证明你掌握模型部署基本链路。

### 项目 2：Jetson 边缘视觉小车感知系统

题目建议：基于 Jetson Orin Nano 的实时目标检测与视觉小车感知系统。

交付内容：

- Jetson 环境搭建记录。
- 摄像头实时检测。
- TensorRT 加速。
- 目标检测结果可视化。
- 设备端 FPS、温度、功耗、延迟记录。
- 可选：结合小车底盘做简单避障或目标跟随。

这个项目用于证明你能把模型真正部署到边缘设备上。

## 8 周路线

### 第 1 周：补基础，建立学习仓库

目标：能在 Windows + Linux/WSL 环境下写脚本、跑模型、记录日志。

- Day 1：整理环境、GitHub 仓库、学习计划、Jetson 采购清单。
- Day 2：Linux 文件、目录、权限、进程、网络命令。
- Day 3：Python 环境管理、虚拟环境、pip、requirements。
- Day 4：C++ 基础补课：输入输出、函数、数组、指针、引用。
- Day 5：PyTorch 推理流程：模型加载、图片预处理、后处理。
- Day 6：性能指标：延迟、FPS、吞吐、warmup、平均值和 P95。
- Day 7：周复盘，完成一个分类模型本地推理 demo。

### 第 2 周：ONNX 和 ONNX Runtime

目标：从 PyTorch 导出 ONNX，并用 ONNX Runtime 推理。

- 导出分类模型到 ONNX。
- 用 Netron 查看模型结构。
- 处理输入 shape、opset、动态 batch。
- 写 benchmark 脚本。
- 完成 PyTorch vs ONNX Runtime 对比。

### 第 3 周：TensorRT 入门到可用

目标：能把 ONNX 转成 TensorRT engine，并完成 FP16 加速。

- 学 trtexec 常用参数。
- 跑 FP32 和 FP16 engine。
- 写 TensorRT Python 推理脚本。
- 记录常见报错。
- 输出第一版 TensorRT 性能报告。

### 第 4 周：项目 1 冲刺

目标：完成 PC 端 YOLO 部署项目。

- 选择 YOLO 小模型。
- 跑通图片和视频检测。
- 完成 ONNX 导出。
- 完成 TensorRT 推理。
- 做性能表格和 README。
- 录一段演示视频或保存检测结果图。

### 第 5 周：Jetson 到手与环境搭建

目标：Jetson 能稳定运行，并跑通摄像头。

- 刷 JetPack。
- 检查 CUDA、cuDNN、TensorRT。
- 安装 jtop。
- 测试摄像头。
- 跑官方 sample。
- 记录温度、功耗和性能基线。

### 第 6 周：Jetson TensorRT 实时检测

目标：把项目 1 移植到 Jetson，并做到实时摄像头检测。

- 转 Jetson 端 TensorRT engine。
- 跑摄像头实时检测。
- 调整输入分辨率。
- 比较 FP32 / FP16。
- 记录 FPS、温度、功耗。
- 学 DeepStream 基础 pipeline。

### 第 7 周：项目 2 冲刺

目标：完成 Jetson 边缘视觉小车感知项目。

- 搭建项目结构。
- 整合摄像头、模型、可视化、日志。
- 可选接入小车控制逻辑。
- 做稳定性测试。
- 完成项目 README。
- 整理项目亮点和简历描述。

### 第 8 周：就业与竞赛准备

目标：把成果变成可投递材料。

- 整理 GitHub 首页。
- 完成 2 个项目 README。
- 写简历项目经历。
- 整理 30 个目标岗位 JD。
- 准备 TensorRT、ONNX、Linux、C++、深度学习面试题。
- 选择 1 个竞赛方向：华为 ICT / 天池 / Kaggle / 嵌入式竞赛。
