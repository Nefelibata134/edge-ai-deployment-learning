# GitHub 学习记录规范

## 仓库建议

仓库名建议：

- `model-deployment-summer`
- `edge-ai-deployment-learning`
- `jetson-vision-deployment`

如果你想找工作展示，建议公开仓库。

## 每天提交什么

每天至少提交一次：

- 学习日志：`logs/dayXX.md`
- 代码实验：`experiments/`
- 项目代码：`projects/`
- 性能报告：`reports/`
- 排错记录：`troubleshooting/`

## commit message 格式

建议用简单英文：

- `docs: add day01 learning log`
- `feat: add pytorch inference demo`
- `bench: compare onnxruntime latency`
- `fix: resolve tensor shape mismatch`
- `report: add tensorrt fp16 benchmark`

## 每周整理

每周末做一次：

- 更新 README。
- 汇总本周成果。
- 整理遇到的问题。
- 保存关键截图或表格。
- 写下下周计划。

## GitHub 首页要展示什么

- 你的方向：Edge AI / Model Deployment / Jetson / TensorRT。
- 两个核心项目链接。
- 技术栈：Python、C++、Linux、PyTorch、ONNX、TensorRT、OpenCV、Jetson。
- 性能对比图或表格。
- 演示视频/GIF/截图。
